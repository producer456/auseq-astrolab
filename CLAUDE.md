## Hard-won gotchas (2026-06-15, on-device)

- **Crash on launch:** `AudioEngine.start()` must touch `engine.mainMixerNode` before `start()`, or an empty graph asserts `inputNode/outputNode != nullptr`. Fixed.
- **Only Apple's built-in instruments appear:** the host needs the **`inter-app-audio`** entitlement (`Sources/AUSeq.entitlements`, wired via `CODE_SIGN_ENTITLEMENTS`). Without it `AVAudioUnitComponentManager` hides all third-party instrument AUv3s.
- **Intermittent / dropping audio:** out-of-process AUv3s starve the audio thread in **Debug**. `deploy.sh` now builds **Release** — always evaluate audio in Release. If dropouts persist in Release, relax `setPreferredIOBufferDuration(0.005)` (5 ms is aggressive for out-of-process hosting).

## Backlog / known issues

- **KeyLab LCD via MCU = dead end (2026-06-16).** The mkII ignores standard Mackie display SysEx (`F0 00 00 66 <id> 12 …`) — verified on device: AUSeq sends it (Result="Sent") but the screen never changes, at every device ID 0x10–0x15. Its display is Arturia-proprietary; driving it would mean sniffing Arturia's own software MIDI (impractical on iPad). The experimental tester lives at Configuration → "KeyLab LCD" (`KeyLabLCDView`) + `MIDIManager.send`/`destinationNames` (MIDI output path now exists — reusable for motor faders / LED feedback if those turn out to be standard MCU). Revisit only if we capture Arturia's protocol.

- **On/off parameter won't turn off (deferred 2026-06-15).** For some plugins, an on/off parameter shown as a slider turns *on* but won't turn *off* — the plugin appears to treat any value > min as "on" and the host control can't reliably set it back to the off value. Rendering already special-cases `.boolean` (Toggle), string-enum `.indexed` (Menu), and numeric `.indexed` (step-1 slider), but this param isn't being caught by those — likely a plain continuous param semantically used as a switch, or a value-readback/snap-back issue. Next step when revisited: add the temp debug readout back (param `unit`/`min`/`max`/live `val`/`valueStrings`) to see the real metadata and whether setting min actually sticks; then consider a tap-the-value toggle that forces min/max regardless of how the plugin tags the param.

## Architecture / conventions

- `AudioEngine` (`@MainActor`): one `AVAudioMixerNode` per track (volume/pan) → main mixer; instruments instantiated with `.loadOutOfProcess`; MIDI to a plugin via `AUAudioUnit.scheduleMIDIEventBlock` (this path is the critical thing to validate on device first).
- `MIDIManager`: CoreMIDI, connects all sources, parses MIDI 1.0 UMP from `MIDIEventList`, posts `MIDIMessage` on the main queue via `onMessage`.
- `AppModel` (`@MainActor`): tracks, selection, routes keyboard + MIDI-in to the selected track; `selectedAU` exposes the hosted `AUAudioUnit`.
- UI: `ParameterListView` rebuilds via `.id(track.id)` on track switch; `AUPluginUIView` embeds the plugin's own `requestViewController` UI.
- Keep new audio/MIDI types `@MainActor` unless they must run on a realtime thread; hop to main before touching `@Published`.
